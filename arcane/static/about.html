Project Compendium
A·R·C·A·N·E
Advanced Roleplay Chat & Anime Network Engine
A character-focused AI roleplay platform. Fifteen anime characters, each with lore-anchored personalities — not chatbots with names, but constructed identities that hold across a full conversation.
The Story

It started from a simple frustration — every AI interaction felt like reading a spreadsheet. Generic. Flat. No soul. ARCANE was built to reverse that.

The goal was to build something that felt like stepping into a favourite anime — where characters respond with actual personality, not just the right words. Each of the fifteen characters has a deep psychological profile, unique speech patterns, and lore-specific context baked into the system prompt.

What started as a single Flask endpoint became a full PWA with a three-tier LLM fallback chain, live web search via tool calling, per-user auth, and a visual novel overlay that breathes alongside the conversation.

FAQ
How does the LLM fallback chain work?
+

Every chat request first checks if the local Ollama tunnel is alive. If it responds, the local model handles inference — fast and private. If the tunnel is down or Ollama fails, the request falls to Groq's Llama 3.3 70b, which also handles web search tool calls natively. Gemini is the last resort. All three tiers share the same character system prompt so personality stays consistent regardless of which model responds.

Is my conversation data stored anywhere?
+

Chat history is stored in your browser's IndexedDB locally and optionally synced to the backend database tied to your account. The backend never logs message contents beyond what's needed for history retrieval. Incognito mode keeps everything in-memory and wipes on session end.

What is the Visual Novel mode?
+

Every assistant response optionally triggers a VN overlay — a character sprite with a typewriter-style dialogue box, background matched to the character's universe, and speed-line flash effects for dramatic messages. Fully client-side, pure CSS and JS, zero external dependencies.

Can I create my own characters?
+

Yes. The custom character builder is a 5-step modal — identity, appearance, personality traits, a live Groq-generated preview, and confirmation. Custom characters are stored in IndexedDB and work identically to pre-loaded ones including tool calling and incognito mode.
